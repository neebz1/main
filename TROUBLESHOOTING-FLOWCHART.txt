FIX A: Cursor Won't Open
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. Check if installed:
   ls -la /Applications/Cursor.app

2. If not installed:
   → Download from cursor.sh
   → Drag to Applications folder

3. If installed but won't open:
   → Right-click → Open (Mac security)
   → Try from terminal: open -a Cursor


FIX A.1: Cursor Opens But Errors Everywhere
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. Restart Cursor completely:
   ⌘Q → Wait 5 seconds → Reopen

2. Check for updates:
   Menu → Check for Updates → Install

3. Clear cache:
   ⌘⇧P → "Clear Editor History"

4. Still broken?
   → Read ZERO-ISSUES-SETUP.md
   → Run ./verify-setup.sh


FIX B: ⌘K Does Nothing
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. Make sure you're in a code file:
   → Create new file: test.js
   → Try ⌘K again

2. Check AI is enabled:
   → ⌘, (settings)
   → Search "AI"
   → Enable "Cursor Tab" and "AI Features"

3. Try alternate: ⌘L
   → Opens chat instead
   → If this works, ⌘K should too

4. Restart Cursor:
   ⌘Q → Reopen → Try again

3. Subscribe to Cursor Pro:

Error: Cline says "No API key"
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. Get API key:
   → OpenRouter: openrouter.ai
   → Anthropic: anthropic.com

2. Add to Cursor:
   → ⌘, → Search "cline"
   → Paste your API key

3. Or add to .env file:
   echo "OPENROUTER_API_KEY=sk-..." > .env
